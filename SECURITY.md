# Security Policy

## Reporting a vulnerability

Report security vulnerabilities privately to [TODO: security contact]. Do not
open a public issue. We aim to acknowledge all reports within 5 business days.

## Disclosure policy

This project follows a 90-day coordinated disclosure standard. We will work with
reporters to understand and remediate confirmed vulnerabilities before public
disclosure. Extensions may be granted by mutual agreement.

## Supported versions

| Version           | Supported |
| ----------------- | --------- |
| main / unreleased | Yes       |

No stable releases exist yet. All security fixes target the main branch.

## Scope

In scope: the linter binary, LSP server, and rule engine.

Out of scope: vendored upstreams. Vulnerabilities in Slang or tree-sitter should
be reported directly to those projects.
